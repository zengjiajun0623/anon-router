01

Create a wallet

No email, no phone, no name. A wallet is a recovery key — hold it and the credits are yours; lose it and no one can help you, which is the point.

recovery key
Save it. Back up wallet stores your key and credits; Import wallet restores them in any browser. Recharge the same wallet and credit adds up. It funds the chat below. To call the API from code or an agent, run the local proxy (quickstart).

session: unlinkable · logs: none

What's private

[x] No signup, no card, no KYC a deposit makes a pseudonymous account; a voucher makes none
[x] No direct link from a spend to your deposit blind-signed ecash (BDHKE)
[x] The provider only ever sees the router, never you requests carry no identity
honest limits — read this part

Blind signatures remove the direct cryptographic link, but unlinkability is only as strong as the crowd claiming and spending in the same window (small, on this alpha). The router still sees amounts and timing. We don't log IPs; the hosting edge can — so connect over the .onion and space out claims and spends. The model reads your prompt (use a local/* model to keep it off third parties). The balance is custodial: we hold the float — keep balances small. The non-custodial escrow, safety core machine-checked in Lean, is on the roadmap. Full posture at /privacy.

Call it from code

There is no hosted API key — a persistent key is a linkable identifier, so that lane was removed. Run the local proxy and point any OpenAI or Anthropic tool at it: Cursor, Claude Code, the SDKs. Full steps: /quickstart.

$ anon-router serve
# proxy listening on http://127.0.0.1:8788
​
$ export OPENAI_BASE_URL=http://127.0.0.1:8788/v1
$ anon-router chat "hello" --model openai/gpt-4o-mini
# pays per request. no key. no account.
