Quickstart

Pay crypto for AI inference, with spends that can't be linked to your deposit. Point any OpenAI- or Anthropic-compatible tool at a local proxy. No account, no card, no code changes.

How it works

You fund once and get blind-signed ecash tokens. A tiny proxy on your machine (anon-router serve) attaches a token to each request and forwards it to the hosted router. The router can verify a token is valid and unspent, but the token carries no account identifier, so it can't link your spend to your deposit. (Without Tor your IP and timing can still correlate your requests, so add --tor for transport privacy.) The proxy is a forwarder, not a model: no GPU, a few MB of RAM.

Why local: the token has to be attached client-side. If a server held it for you, that server would see the link, which is exactly what we remove. So there's a thin local relay.

1 · Install

Needs Python 3.10+. Installs the anon-router CLI from source (a published pip install anon-router is coming). A virtualenv is the always-works path — macOS often has no bare pip:

python3 -m venv anon-router-env
source anon-router-env/bin/activate
pip install git+https://github.com/zengjiajun0623/anon-router.git

Prefer one global command instead? With pipx or uv: pipx install git+https://github.com/zengjiajun0623/anon-router.git or uv tool install git+https://github.com/zengjiajun0623/anon-router.git — both put anon-router on your PATH with no venv to activate.

2 · Get credits

Option A — voucher (no crypto). Redeeming mints ecash straight into your wallet, so funding is done: skip step 3 and go to step 4.

anon-router redeem ar-XXXXXXXX

Option B — deposit Sepolia test ETH from a funding key file (never paste a raw key on the command line):

anon-router deposit 0.05 --key wallet.json

Need Sepolia test ETH? Get it free from a faucet (Google Cloud or pk910 PoW faucet), sent to your funding wallet's address.

3 · Claim to ecash (deposit only)

If you deposited, drain your whole balance into ecash tokens in one step. Voucher users already hold ecash and can skip this. Claiming the full balance (not a partial amount matching your deposit) is what keeps funding and spending unlinkable.

anon-router claim

4 · Run the proxy

anon-router serve
# → base_url = http://127.0.0.1:8788/v1   (api_key = anything)
# add --tor to hide your IP from the router too

Out of ecash later? Fund again, then restart the proxy to claim the new balance (the proxy claims only at startup, on purpose — a per-request claim would be a re-link marker).

5 · Call it

OpenAI SDK / Python chat + tools + streaming

from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8788/v1", api_key="anon")

resp = client.chat.completions.create(
    model="openai/gpt-4o-mini",
    messages=[{"role": "user", "content": "In one word, what is ecash?"}],
)
print(resp.choices[0].message.content)

curl

curl http://127.0.0.1:8788/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4o-mini",
    "messages": [{"role": "user", "content": "In one word, what is ecash?"}]
  }'

Claude Code Anthropic API

export ANTHROPIC_BASE_URL=http://127.0.0.1:8788
export ANTHROPIC_API_KEY=anon      # any non-empty value
claude              # every request now pays private ecash

On the Anthropic lane the answer arrives once it's complete rather than token by token. That's deliberate: the proxy settles each request's ecash before replying, which keeps concurrent agent requests from racing your wallet.

Cursor, aider, LangChain, LiteLLM… anything OpenAI-compatible: set the base URL to http://127.0.0.1:8788/v1 and the API key to any non-empty string.

Models

anon-router models            # list what the router can route to

Use provider-prefixed IDs like openai/gpt-4o-mini or anthropic/claude-3.5-sonnet. Bare names (gpt-4o-mini) are mapped for you. local/* models are free.

Billing is 1 credit = $0.0001, with a 1-credit minimum per request, so very cheap calls (tiny models) round up. Fine for a testnet; real per-token pricing is a mainnet concern.

What's private, honestly

Spends are cryptographically unlinkable to your deposit (blind signatures), and the model provider only ever sees the router. Limits: the model reads your prompt content (use a local/* model to keep it off third parties); the balance is custodial (keep it small); on-chain funding is pseudonymous (fund from a fresh wallet, a common round amount); and without --tor your IP and timing still correlate your requests. Full posture at /privacy.
